# 소리지갑 (Sound Wallet)

> 시각장애인을 위한 **AI 기반 한국 화폐 인식 음성 안내 시스템**  
> 스마트폰 카메라로 촬영한 **지폐·동전**을 인식하고, 금액을 **음성(TTS)** 으로 안내하는 접근성 서비스

---

## Overview

**소리지갑**은 시각장애인과 고령자처럼 시각 정보 접근이 어려운 사용자를 위해 만든  
**한국 화폐 인식 보조 서비스**입니다.

기존 화폐 인식 서비스는 외국 화폐 중심, 동전 미지원, 복잡한 UI, 온라인 의존성이 강한 경우가 많았습니다.  
소리지갑은 이러한 한계를 보완해 다음과 같은 방향으로 설계했습니다.

- **한국 지폐 + 동전 지원**
- **음성 중심 UI**
- **오프라인 사용 가능**
- **전맹 / 고령자 사용자까지 고려한 접근성 설계**

이 프로젝트는 단순한 객체 탐지를 넘어,  
실제 결제 상황에서 사용 가능한 **화폐 인식 → 금액 합산 → 음성 안내** 흐름을 구현하는 데 초점을 맞췄습니다.

---

## Why This Project?

시각장애인은 일상 속에서 다음과 같은 문제를 겪습니다.

- 동전은 촉각만으로 **구분이 어려움**
- 지폐는 마모되면 **촉각 인식이 어려움**
- 기존 앱은 외국 화폐 중심이거나 사용 방식이 복잡함
- 디지털 금융 서비스 접근성이 낮아 **현금 의존도**가 높음

소리지갑은 이러한 현실 문제를 해결하기 위해 시작한 프로젝트입니다.  
기술로 접근성을 높이고, **금전적 자립과 생활 편의 향상**에 기여하는 것을 목표로 했습니다.

---

## Key Features

- **한국 화폐 8종 인식**
  - 지폐: 1,000 / 5,000 / 10,000 / 50,000원
  - 동전: 10 / 50 / 100 / 500원
- **실시간 화폐 탐지 및 분류**
- **여러 화폐 금액 자동 합산**
- **한국어 음성 안내 (TTS)**
- **앱 / 웹 연동**
- **오프라인 사용 가능**

---

## System Flow

```text
스마트폰 카메라 입력
    ↓
YOLOv8n 기반 화폐 탐지/분류
    ↓
인식된 화폐 금액 합산
    ↓
TTS 음성 안내
    ↓
앱 / 웹 인터페이스 제공
````

프로젝트는 스마트폰 카메라로 화폐를 촬영한 뒤,
AI 모델이 화폐 종류를 인식하고, Kotlin에서 금액을 계산한 후,
TTS로 사용자에게 결과를 전달하는 구조로 설계했습니다.

## Model Design Journey

소리지갑의 핵심은 **정확도와 실시간성 사이의 균형**이었습니다.

### 1) YOLOv8 + Crop + ResNet18

초기에는 다음 구조를 검토했습니다.

```text
YOLOv8 (화폐 위치 탐지)
→ Crop
→ ResNet18 (화폐 종류 분류)
```

#### 장점

* 분류 정밀도 향상 가능
* YOLO는 위치 탐지, ResNet은 세부 분류에 집중 가능

#### 한계

* **YOLO → Crop → ResNet**의 2단계 처리로 인해 **추론 속도 저하**
* YOLO가 박스를 잘못 잡으면 **crop 이미지 품질에 따라 분류 정확도 저하**

---

### 2) YOLOv8 단일 모델 구조

이후 탐지와 분류를 한 번에 처리하는 단일 YOLO 구조로 전환했습니다.

```text
YOLOv8
→ 탐지 + 8클래스 분류
→ 결과 출력
```

#### 장점

* 속도 향상
* crop 품질 의존 문제 감소
* 배포 단순화

#### 한계

* 작은 객체인 **동전 인식 성능 저하**
* 유사한 화폐 클래스 간 혼동 가능성

---

### 3) 최종 구조: 동전 모델 + 지폐 모델 분리

최종적으로는 다음과 같은 **이중 모델 구조**를 채택했습니다.

```text
YOLOv8 Coin Model
+
YOLOv8 Money Model
→ 결과 결합 및 신뢰도 기반 판별
```

#### 최종 선택 이유

* 동전은 작은 객체라 별도 특화 모델이 더 유리
* 전체 화폐 모델은 지폐 중심 인식에 강점
* 두 모델의 결과를 결합해 **성능을 상호 보완**

즉,
**실시간성은 유지하면서도 작은 객체 인식 성능을 보완하는 방향**으로 모델 구조를 개선했습니다.

## Performance

### Quantitative Results

* **Recall: 0.98+**
* **mAP50: 0.99+**
* **cls_loss: 6.0 → 1.0 이하 안정 수렴**

### Interpretation

* 대부분의 화폐를 **놓치지 않고 탐지**
* 화폐 위치를 **정확하게 탐지**
* 학습이 안정적으로 수렴하며 **과적합 없이 성능 확보**

### Confusion Matrix Insight

* 대부분의 화폐 클래스에서 높은 정확도
* 일부 **100원 / 50,000원** 간 소수 혼동 발생
* background 클래스와의 혼동은 거의 없음

즉,
실제 서비스 적용을 고려했을 때 **높은 수준의 탐지 성능과 안정성**을 확보했습니다.
